Spring Boot
Spring Boot is a framework built on top of Spring that helps you create applications quickly with minimal configuration.

✔ No XML configuration
✔ Embedded server (Tomcat)
✔ Auto-configuration
✔ Faster development

Key Goals of Spring Boot
✅ Reduce configuration
✅ Enable fast development
✅ Provide production-ready features
 
1. Auto-Configuration
👉 Automatically configures beans based on dependency
✔ No manual setup needed
2. Embedded Server
👉 No need to install Tomcat separately
✔ App runs directly
3. Starter Dependencies
👉 Ready-made dependencies
✔ No need to add multiple libraries manually
4. Production Ready (Actuator)
👉 Monitoring features
5. Minimal Configuration
👉 Mostly configured via: application.properties

Spring vs SpringBoot
Spring requires manual configuration and setup, whereas Spring Boot simplifies development using auto-configuration, embedded servers, and starter dependencies.

Architecture Flow

Main Class (@SpringBootApplication)
        ↓
Auto Configuration
        ↓
Component Scanning
        ↓
Beans Creation
        ↓
Embedded Server (Tomcat)
        ↓
Application Starts

3. Starter Dependencies
👉 Starter dependencies are pre-configured dependency packages that include everything needed for a feature.

With starters:
One dependency → all included

spring-boot-starter-web ->Spring MVC, Jackson (JSON), Embedded Tomcat
spring-boot-starter-data-jpa
spring-boot-starter-security

Advantages
✅ Saves time
✅ Avoids dependency conflicts
✅ Easy configuration

4. Spring Boot Actuator (Monitoring) :  Spring Boot Actuator helps monitor application health, performance, and system status using ready-made endpoints.
👉 Provides production-ready monitoring features
by adding spring-boot-starter-actuator-> the actuators are enabled

In real applications, we need to know:
✅ Is the app running?
✅ Is DB working?
✅ Memory usage?
✅ CPU usage?
✅ Logs and metrics?
👉 Actuator gives all this automatically ✅

1. Health
http://localhost:8080/actuator/health-> ✔ Checks if application is running
output->
{"status": "UP"}

2. Metrics
/actuator/metrics-> ✔ Shows app performance (memory, CPU, etc.)
 3. Info
/actuator/info-> ✔ App information
4. Beans
/actuator/beans-> ✔ List of all Spring beans
5. Environment
/actuator/env-> ✔ Environment properties


Application starts
       ↓
Actuator loads monitoring endpoints
       ↓
Expose URLs (/actuator/*)
       ↓
User / DevOps can monitor app
Logging in Spring Boot->  Spring Boot provides built-in logging support to track application behavior, errors, and execution flow.
👉 Spring Boot provides built-in logging support
Spring Boot uses:
✅ Logback (default)
✅ With SLF4J (logging API)
In real applications:
✅ Debug issues
✅ Track user actions
✅ Monitor system behavior
✅ Identify errors

Level	Purpose
TRACE	Very detailed logs
DEBUG	Debugging
INFO	Normal flow ✅
WARN	Warnings
ERROR	Errors

 Configure Logging (application.properties)

logging.level.root=INFO
logging.level.com.myapp=DEBUG
 Root → overall log level
Package-specific logging

Logging in Spring Boot

   ↓ log.info()
Service
   ↓ log.debug()
Repository
   ↓ log.error()
