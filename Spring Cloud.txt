Spring Cloud👉 Spring Cloud provides tools and frameworks to build and manage microservices in distributed systems. 1. Microservices Architecture👉 Microservices architecture divides an application into smaller, independent services that communicate via APIs.👉 Each service:Runs independently ✅Has its own database ✅Communicates via REST APIs ✅✅ Scalability✅ Independent deployment✅ Fault isolation✅ Faster development 2. Service Discovery👉 Service discovery allows services to find and communicate with each other dynamically using a registry.👉 Microservices run on different ports/IPs👉 Instead of hardcoding URLs ❌👉 Use service names ✅Service registers → Registry (Eureka)        ↓Client service → asks registry        ↓Gets service location → communicates1. Eureka (Service Registry):👉 Eureka is a service registry used in Spring Cloud for registering and discovering microservices.👉 Services communicate using names:USER-SERVICEORDER-SERVICE✅ No need to remember IP/port ✅✅ Dynamic discovery ✅1. Microservice starts2. Registers itself with Eureka Server3. Eureka stores service details (name, IP, port)4. Another service requests a service location5. Eureka provides service details6. Service-to-service communication happensOrder Service → asks Eureka → "Where is USER-SERVICE?"           ↓Eureka → returns service instance (IP/port)           ↓Order Service → calls User Service ✅Eureka Server👉 Eureka Server is a registry that maintains a list of all available microservices and enables service discovery in a microservices architecture.👉 Acts as:✅ Service registry (stores services)✅ Discovery server (helps find services)✅ Health monitor (tracks services)@EnableEurekaServer2.Eureka Client:👉 Eureka Client is a Spring Boot application that registers with the Eureka Server and enables service discovery to communicate with other microservices👉 A Eureka Client performs 2 main functions✅ Register itself to Eureka Server✅ Discover other services from EurekaMicroservice startsRegisters with Eureka ServerSends heartbeat (alive signal)Queries registry to find other servicesCommunicates using service namesUser Service → registers → Eureka ServerOrder Service → queries → Eureka for User ServiceOrder Service → calls USER-SERVICE ✅@EnableDiscoveryClientAPI Gateway:API Gateway is a centralized entry point that receives all client requests and routes them to appropriate microservices.Spring Cloud Gateway (API Gateway):Spring Cloud Gateway is an API Gateway that acts as a single entry point for all client requests in a microservices architecture.👉 Acts as:✅ Request Router✅ Security Layer✅ Load Balancer✅ Filter/InterceptorClient Request↓API Gateway↓Route to correct microservice↓Response sent backRouting👉 Gateway routes requests to servicesPredicates👉 Conditions to match requestsFilters ⭐👉 Used to modify request/responseLoad Balancing ⭐👉 Automatically distributes requestsSecurity Integration ⭐👉 Gateway can handle:✅ Authentication (JWT validation)✅ Authorization✅ Rate limitingConfig Server (central config)Config Server is a centralized configuration service that stores and manages configuration for multiple microservices in one place.✅ Centralized management ✅✅ Easy updates ✅✅ No duplication ✅Config stored in Git repoConfig Server reads configMicroservices fetch config at startupServices use config dynamicallyUser Service startup↓Calls Config Server↓Gets application config↓Runs with config ✅@EnableConfigServeFeign Client (service-to-service calls):Feign Client is a declarative REST client used in Spring Cloud to call other microservices easily using interfaces.✅ Clean code ✅✅ No manual HTTP calls ✅✅ Uses service name ✅Service A → calls Feign Client↓Feign Client → calls Service B↓Response returned@EnableFeignClientsRESTTEMPLATE:RestTemplate is used in Spring to make HTTP calls (GET, POST, PUT, DELETE) to other services or APIs.Swagger:Swagger (OpenAPI) is a tool used to document and test REST APIs.Swagger is used to generate interactive API documentation where developers can view and test REST endpoints✅
